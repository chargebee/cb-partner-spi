# OpenPay SPI Integration

The **OpenPay Service Provider Interface (SPI)** enables seamless communication between Chargebee and external payment gateways through a standardized interface.

You can refer to the spec file for this SPI [here](https://github.com/chargebee/cb-partner-spi/blob/main/spec/spi/openapi_openpay.yml).

Chargebee leverages external payment services to execute the following operations efficiently:

### 1. Payment Method Management
Create, update, retrieve, confirm, and delete payment methods, along with associated customer profiles, in the payment gateway.

### 2. Payment Processing
Process, verify, capture, refund, and void payments through the configured gateway.

### 3. Payment Intents
Create payment intents to prepare for payment transactions, including setting up amount, currency, and customer details.

### 4. Gateway Setup
Set up and validate gateway account credentials, supporting both direct gateway integrations and orchestrator-based setups.

## The Role of the OpenPay Adapter

Chargebee maintains a built-in adapter layer that connects to a range of supported payment gateways out of the box. Most merchants do not need to implement anything—Chargebee handles the communication with the payment gateway on their behalf.

## Building a Custom OpenPay Adapter

In some cases, you may want to go beyond the built-in adapter. Implementing the OpenPay SPI lets you build a custom adapter if any of the following apply:

- **Unsupported Gateway:**
  The payment gateway you want to use is not yet supported by Chargebee. You can implement an adapter yourself to bring that gateway into the Chargebee ecosystem.

- **Payment Gateway Provider:**
  You are a gateway provider and want to offer a native, certified integration with Chargebee for your merchants.

- **Custom Adapter for an Existing Gateway:**
  You need behavior that differs from Chargebee's built-in adapter—such as custom routing logic, orchestrator-specific handling, or additional gateway parameters.

- **System Integrator:**
  You are building a connector on behalf of a merchant or gateway, bridging a payment service with Chargebee.
